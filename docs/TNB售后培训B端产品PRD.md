# TNB 售后学练平台 B 端产品需求文档

## 1. 需求背景

### 1.1 业务背景

TNB 售后体系覆盖服务代表、机电技师、钣金技师、喷漆技师、保修员、配附件专员、车间负责人、技术负责人、质检员等多个岗位，每个岗位分初级、中级、高级、管理四个认证等级。目前售后培训依赖线下培训和人工出题，存在以下问题：

- 课件分散在 PDF、PPT、视频等多种格式中，学员检索困难
- 考点和题目依赖人工维护，更新效率低
- 不同岗位、不同等级的培训内容缺乏系统化关联
- 学员练习数据无法追踪，培训效果难以衡量

### 1.2 产品定位

TNB 售后学练平台是面向售后培训场景的智能化学习练习系统，通过 AI 技术实现课件知识切片、考点自动抽取、智能出题，支持岗位课程（认证等级导向）和专项课程（知识模块导向）两种培训模式，为售后人员提供个性化的练习推送和学习效果追踪。

### 1.3 目标用户

| 角色 | 职责 | 使用频率 |
|------|------|----------|
| 系统管理员 | 维护能力维度、岗位课程、系统配置 | 低 |
| 培训师 | 维护专项课程、上传课件、抽取考点、AI 出题、审核发布 | 高 |
| 学员 | 接收推送练习、自主练习、查看练习记录 | 高 |

### 1.4 项目范围

**MVP 范围**：专项课程完整闭环（课程创建 → 课件上传 → 知识切片 → 考点抽取 → AI 出题 → 人工审核 → 发布 → 学员练习 → 数据查看）。岗位课程在 MVP 中仅展示后台已维护的结果，支持题目微调，暂不支持重新出题。

## 3. 功能需求详述

### 3.1 课程管理

#### 3.1.1 功能概述

课程分为**岗位课程**和**专项课程**两类，分开维护。岗位课程面向认证等级，一个认证等级对应一个课程；专项课程面向知识模块，区分技术类和非技术类。

#### 3.1.2 岗位课程

**业务逻辑**：
- 岗位课程数据从现有系统同步，管理员后台维护
- 一个岗位等级对应一个课程（如 JSR 服务代表初级对应一个课程）
- 课程包含多个能力维度，每个能力维度包含多个知识模块
- 岗位课程在 MVP 中仅展示结果，不支持重新出题，但支持题目手动微调

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 课程 ID | 字符串 | 系统生成 | 唯一标识 |
| 课程名称 | 字符串 | 是 | 如"服务代表初级（JSR）" |
| 岗位类型 | 枚举 | 是 | 技术 / 非技术 |
| 岗位名称 | 枚举 | 是 | 服务代表、机电技师等 |
| 认证等级 | 枚举 | 是 | 初级 / 中级 / 高级 / 管理 |
| 认证 Code | 字符串 | 是 | 如 JSR-C、ISR-C |
| 课程状态 | 枚举 | 是 | 上架 / 下架 |
| 创建时间 | 时间 | 系统自动 | |
| 更新时间 | 时间 | 系统自动 | |

**交互逻辑**：
- 岗位课程列表页：展示所有岗位课程，支持按岗位类型、岗位名称、认证等级筛选
- 课程详情页：展示课程下的能力维度 → 知识模块 → 考点 → 题库，只读展示
- 题目微调：在题库列表中点击"编辑"，可修改题干、选项、解析

#### 3.1.3 专项课程

**业务逻辑**：
- 专项课程由培训师创建和维护
- 一个专项课程只有一个能力维度（"知识掌握"），课程下直接挂知识模块
- 专项课程区分技术类和非技术类
- 课程下架后，C 端该课程的练习也下架

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 课程 ID | 字符串 | 系统生成 | 唯一标识 |
| 课程名称 | 字符串 | 是 | 如"ES9 新车型技术培训" |
| 课程类型 | 枚举 | 是 | 技术 / 非技术 |
| 课程状态 | 枚举 | 是 | 上架 / 下架 |
| 创建人 | 字符串 | 系统自动 | 培训师域账号 |
| 创建时间 | 时间 | 系统自动 | |
| 更新时间 | 时间 | 系统自动 | |

**交互逻辑**：
- 专项课程列表页：展示培训师有权限的专项课程，支持按课程类型、状态筛选
- 新建课程：填写课程名称、选择课程类型（技术/非技术）
- 编辑课程：修改课程名称、课程类型、上下架状态
- 课程详情页：展示课程下的知识模块列表，可进入知识模块管理

#### 3.1.4 页面原型

```
+--------------------------------------------------+
| 课程管理                              [+ 新建课程] |
+--------------------------------------------------+
| 标签页: [岗位课程] [专项课程]                      |
+--------------------------------------------------+
| 筛选: [岗位类型: 全部 ▼] [岗位: 全部 ▼] [等级: 全部 ▼] |
+--------------------------------------------------+
| 课程名称          | 岗位类型 | 岗位     | 等级   | 状态   | 操作 |
|-------------------|----------|----------|--------|--------|------|
| 服务代表初级(JSR) | 非技术   | 服务代表 | 初级   | 上架   | 查看 |
| 机电技师中级(ST)  | 技术     | 机电技师 | 中级   | 上架   | 查看 |
| ...               |          |          |        |        |      |
+--------------------------------------------------+
```

### 3.2 能力维度管理

#### 3.2.1 功能概述

能力维度是岗位课程的特有层级，一个岗位课程包含多个能力维度（如"L1-专业知识"、"L2-标准服务流程执行"），每个能力维度包含多个知识模块。能力维度由管理员后台维护，基本不会变动。

#### 3.2.2 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 能力维度 ID | 字符串 | 系统生成 | 唯一标识 |
| 能力维度名称 | 字符串 | 是 | 如"L1-专业知识" |
| 能力维度描述 | 文本 | 否 | 能力要求说明 |
| 所属课程 ID | 字符串 | 是 | 关联岗位课程 |
| 排序 | 整数 | 是 | 展示顺序 |

#### 3.2.3 交互逻辑

- 能力维度列表：按课程分组展示能力维度
- 新建/编辑：管理员维护能力维度名称、描述、排序
- 删除：能力维度下无知识模块时才允许删除

### 3.3 知识模块管理

#### 3.3.1 功能概述

知识模块是课程下的学习单元，对应具体的课件集合。

**岗位课程**：知识模块挂在能力维度下，一个能力维度包含多个知识模块。
**专项课程**：知识模块直接挂在课程下，课程只有一个能力维度（"知识掌握"）。

#### 3.3.2 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 知识模块 ID | 字符串 | 系统生成 | 唯一标识 |
| 知识模块名称 | 字符串 | 是 | 如"车辆解锁、上锁和门窗控制" |
| 所属课程 ID | 字符串 | 是 | 关联课程 |
| 所属能力维度 ID | 字符串 | 岗位课程必填 | 关联能力维度 |
| 排序 | 整数 | 是 | 展示顺序 |
| 创建时间 | 时间 | 系统自动 | |

#### 3.3.3 交互逻辑

- 知识模块列表：按课程（或能力维度）分组展示
- 新建/编辑：维护知识模块名称、排序
- 删除：知识模块下无课件时才允许删除
- 进入知识模块：可管理课件、考点、题库

### 3.4 课件知识管理

#### 3.4.1 功能概述

支持 PDF、PPT、视频三种格式的课件上传，系统进行知识切片和向量化存储，供 C 端 ChatBot 检索使用。

#### 3.4.2 课件上传

**支持的格式**：
- PDF：直接解析文本内容
- PPT：解析每页文本内容
- 视频：提取音频转文字（ASR）

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 课件 ID | 字符串 | 系统生成 | 唯一标识 |
| 课件名称 | 字符串 | 是 | 原始文件名或自定义名称 |
| 课件类型 | 枚举 | 是 | PDF / PPT / 视频 |
| 所属知识模块 ID | 字符串 | 是 | 关联知识模块 |
| 文件 URL | 字符串 | 是 | 存储地址 |
| 切片状态 | 枚举 | 系统自动 | 待切片 / 切片中 / 已切片 / 切片失败 |
| 创建时间 | 时间 | 系统自动 | |

#### 3.4.3 知识切片

**切片策略**：
- PDF：按页切片 + AI 语义切片结合
- PPT：按页切片
- 视频：按 ASR 文本的语义段落切片

**切片字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 切片 ID | 字符串 | 唯一标识 |
| 所属课件 ID | 字符串 | |
| 切片内容 | 文本 | 文本内容 |
| 切片类型 | 枚举 | 按页 / 按章节 / 语义段落 |
| 页码/时间段 | 字符串 | PDF/PPT 页码或视频时间段 |
| 向量 Embedding | 向量 | 向量化表示 |

**交互逻辑**：
- 课件上传后，系统异步进行切片和向量化
- 切片完成后，可查看切片列表和切片内容
- 切片失败的课件可重新触发切片
- C 端 ChatBot 基于向量化存储的切片内容回答学员问题

#### 3.4.4 页面原型

```
+--------------------------------------------------+
| 课件管理                                [+ 上传课件] |
+--------------------------------------------------+
| 知识模块: 车辆解锁、上锁和门窗控制                   |
+--------------------------------------------------+
| 课件名称          | 类型 | 大小  | 切片状态 | 操作   |
|-------------------|------|-------|----------|--------|
| ES9_NT3_用户手册  | PDF  | 15MB  | 已切片   | 查看切片 |
| 门窗控制培训视频  | 视频 | 120MB | 切片中   | -      |
+--------------------------------------------------+
```

### 3.5 考点管理

#### 3.5.1 功能概述

考点是针对一个知识点的标准问答/概念解释，从课件切片中 AI 自动抽取，经人工审核后用于 AI 出题。

#### 3.5.2 考点抽取

**业务逻辑**：
- 课件切片完成后，管理员/培训师手动点击"开始抽取考点"
- 系统调用 AI 模型，基于考点抽取 Prompt 模板从切片中提取考点
- 抽取的考点进入"待审核"状态
- 一个考点只来源于一个切片
- 考点继承所属课程的分类（技术/非技术）

**考点字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 考点 ID | 字符串 | 系统生成 | 唯一标识 |
| 考点内容 | 文本 | 是 | 知识点描述/问答 |
| 所属切片 ID | 字符串 | 是 | 来源切片 |
| 所属知识模块 ID | 字符串 | 是 | |
| 难度 | 枚举 | 是 | 易 / 中 / 难 |
| 状态 | 枚举 | 是 | 待审核 / 已通过 / 已拒绝 |
| 创建时间 | 时间 | 系统自动 | |

#### 3.5.3 考点审核

**交互逻辑**：
- 考点列表：展示待审核、已通过、已拒绝的考点，支持按状态、难度筛选
- 审核操作：
  - 通过：考点状态变为"已通过"，可用于 AI 出题
  - 拒绝：考点状态变为"已拒绝"，可填写拒绝原因
  - 编辑：修改考点内容、难度
- 批量审核：支持批量通过/拒绝

#### 3.5.4 页面原型

```
+--------------------------------------------------+
| 考点管理                              [开始抽取考点] |
+--------------------------------------------------+
| 筛选: [状态: 待审核 ▼] [难度: 全部 ▼]               |
+--------------------------------------------------+
| 考点内容                    | 来源课件 | 难度 | 状态 | 操作 |
|-----------------------------|----------|------|------|------|
| ES9 车辆解锁方式有哪些？    | ES9手册  | 中   | 待审核 | [通过] [拒绝] [编辑] |
| 门窗控制的安全机制是什么？  | ES9手册  | 难   | 待审核 | [通过] [拒绝] [编辑] |
+--------------------------------------------------+
```

### 3.6 Prompt 模板管理

#### 3.6.1 功能概述

Prompt 模板用于定义 AI 在考点抽取和出题环节的行为规范，包括出题方向、难度定义、禁止规则等。

#### 3.6.2 模板类型

| 用途 | 说明 |
|------|------|
| 考点抽取 | 定义从课件切片中提取考点的规则 |

#### 3.6.3 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 模板 ID | 字符串 | 系统生成 | 唯一标识 |
| 模板名称 | 字符串 | 是 | 如"技术类-新车型出题模板" |
| 模板用途 | 枚举 | 是 | 考点抽取 / 出题 |
| 适用分类 | 枚举 | 是 | 技术 / 非技术 |
| Prompt 内容 | 文本 | 是 | 完整的 Prompt 文本 |
| 创建人 | 字符串 | 系统自动 | |
| 创建时间 | 时间 | 系统自动 | |

#### 3.6.4 交互逻辑

- 模板列表：按用途和适用分类分组展示
- 新建/编辑：维护模板名称、用途、适用分类、Prompt 内容
- 删除：模板未被使用时允许删除
- 岗位课程的 Prompt 模板编辑权限仅给管理员
- 培训师可维护专项课程的 Prompt 模板

### 3.7 AI 出题管理

#### 3.7.1 功能概述

基于已审核通过的考点，选择 Prompt 模板，由 AI 自动生成题目，经人工审核后发布到题库。

#### 3.7.2 出题流程

```
选择考点（最多 3 个）→ 选择 Prompt 模板 → AI 生成题目 → 人工审核 → 发布到题库
```

**业务规则**：
- 每次最多选择 3 个考点
- 每个考点只能出 1 道题
- 题型固定：单选、多选、判断
- 难度可定义：难、中、易（在 Prompt 中约束）
- 审核不通过的题目可直接删除
- 已发布的题目不能删除，只能失效

#### 3.7.3 题目字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 题目 ID | 字符串 | 系统生成 | 唯一标识 |
| 题干 | 文本 | 是 | 题目内容 |
| 题型 | 枚举 | 是 | 单选 / 多选 / 判断 |
| 选项 | JSON | 是 | 选项列表，含正确标记 |
| 正确答案 | 字符串 | 是 | 正确选项标识 |
| 解析 | 文本 | 是 | 答案解析 |
| 关联考点 IDs | 数组 | 是 | 关联的考点 |
| 难度 | 枚举 | 是 | 易 / 中 / 难 |
| 所属知识模块 ID | 字符串 | 是 | |
| 状态 | 枚举 | 是 | 待审核 / 已通过 / 已发布 / 已失效 |
| 创建时间 | 时间 | 系统自动 | |

#### 3.7.4 审核与微调

**交互逻辑**：
- AI 生成的题目进入"待审核"状态
- 审核操作：
  - 通过：题目状态变为"已通过"，可继续发布
  - 删除：直接删除题目
  - 编辑：修改题干、选项、解析（不可修改题型）
- 重新出题：对不满意的题目，可调整 Prompt 模板后重新生成
- 发布：审核通过的题目发布到对应知识模块的题库

#### 3.7.5 页面原型

```
+--------------------------------------------------+
| AI 出题                                             |
+--------------------------------------------------+
| 步骤 1: 选择考点 (最多 3 个)                         |
| [x] ES9 车辆解锁方式有哪些？                        |
| [x] 门窗控制的安全机制是什么？                      |
| [ ] NOMI 语音控制功能介绍                           |
+--------------------------------------------------+
| 步骤 2: 选择 Prompt 模板                            |
| [技术类-新车型出题模板 ▼]                           |
+--------------------------------------------------+
| [开始出题]                                          |
+--------------------------------------------------+
| 生成结果:                                           |
| 题干: ES9 支持以下哪些解锁方式？                    |
| 题型: 多选                                          |
```

### 3.8 题库管理

#### 3.8.1 功能概述

题库按课程-知识模块维度组织，管理已发布的题目。

#### 3.8.2 业务规则

- 专项课程的题目发布维度：针对一个课程下的一个知识模块发布
- 已发布的题目不能删除，只能取消发布（失效）
- 取消发布后，C 端用户看不到该题目
- 用户已练习的数据快照了题目 ID
- 题目不需要版本管理
- 岗位课程的题目支持手动微调（编辑题干、选项、解析）

#### 3.8.3 交互逻辑

- 题库列表：按课程 → 知识模块层级展示题目
- 筛选：按课程、知识模块、题型、难度、状态筛选
- 编辑：修改题干、选项、解析（仅未发布或本模块内的题目）
- 取消发布：已发布的题目可取消发布，状态变为"已失效"
- 重新发布：失效的题目可重新审核后发布

#### 3.8.4 页面原型

```
+--------------------------------------------------+
| 题库管理                                            |
+--------------------------------------------------+
| 筛选: [课程: ES9新车型培训 ▼] [知识模块: 全部 ▼]     |
|       [题型: 全部 ▼] [难度: 全部 ▼] [状态: 全部 ▼]   |
+--------------------------------------------------+
| 题干                    | 题型 | 难度 | 状态   | 操作 |
|-------------------------|------|------|--------|------|
| ES9 支持哪些解锁方式？  | 多选 | 中   | 已发布 | [编辑] [取消发布] |
| 门窗安全机制是什么？    | 单选 | 难   | 已发布 | [编辑] [取消发布] |
+--------------------------------------------------+
```

### 3.9 发布管理

#### 3.9.1 功能概述

将一个知识模块的题目配置发布范围，发布后学员通过飞书机器人收到练习通知。

#### 3.9.2 发布配置

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 发布 ID | 字符串 | 系统生成 | 唯一标识 |
| 所属课程 ID | 字符串 | 是 | |
| 所属知识模块 ID | 字符串 | 是 | 发布的知识模块 |
| 发布范围类型 | 枚举 | 是 | 岗位类型 / 岗位 / 用户 |
| 发布范围值 | 数组 | 是 | 具体的技术/非技术、岗位名称、用户域账号 |
| 练习截止时间 | 时间 | 否 | 过期后 C 端置灰提示 |
| 发布状态 | 枚举 | 是 | 已发布 / 已取消 |
| 发布时间 | 时间 | 系统自动 | |

**发布范围规则**：
- 按岗位类型：发布给所有技术类或非技术类用户
- 按岗位：发布给具体岗位的所有用户（如所有"服务代表"）
- 按用户：发布给指定的具体用户（域账号列表）

#### 3.9.3 交互逻辑

- 发布列表：展示历史发布记录
- 新建发布：
  1. 选择课程和知识模块
  2. 选择发布范围类型和具体值
  3. 设置练习截止时间（可选）
  4. 确认发布
- 取消发布：已发布的配置可取消，C 端置灰提示已过期
- 发布通知：通过飞书机器人推送消息给目标用户

#### 3.9.4 页面原型

```
+--------------------------------------------------+
| 发布管理                                  [+ 新建发布] |
+--------------------------------------------------+
| 课程-知识模块          | 发布范围       | 截止时间 | 状态 | 操作 |
|------------------------|----------------|----------|------|------|
| ES9培训-车辆解锁控制   | 技术类岗位     | 2026-07-01 | 已发布 | [取消] |
| ES9培训-补能和供电     | 服务代表,机电  | 2026-07-15 | 已发布 | [取消] |
+--------------------------------------------------+
```

## 4. 数据模型

### 4.1 核心实体关系

```
岗位 (Position)
  ├── 岗位类型: 技术 / 非技术
  ├── 岗位名称: 服务代表、机电技师...
  └── 认证等级: 初级 / 中级 / 高级 / 管理

课程 (Course)
  ├── 岗位课程: 关联岗位 + 认证等级
  └── 专项课程: 课程类型(技术/非技术)

能力维度 (Competency) [仅岗位课程]
  └── 关联课程

知识模块 (KnowledgeModule)
  ├── 岗位课程: 关联能力维度
  └── 专项课程: 直接关联课程

课件 (Courseware)
  └── 关联知识模块

切片 (Slice)
  └── 关联课件

考点 (ExamPoint)
  └── 关联切片 + 知识模块

Prompt 模板 (PromptTemplate)
  ├── 用途: 考点抽取 / 出题
  └── 适用分类: 技术 / 非技术

题目 (Question)
  └── 关联考点 + 知识模块

发布配置 (PublishConfig)
  └── 关联课程 + 知识模块 + 发布范围

练习记录 (PracticeRecord)
  └── 关联用户 + 题目 + 课程 + 知识模块
```

## 5. 非功能需求

### 5.1 性能需求

- 课件上传后切片完成时间：PDF/PPT < 5 分钟，视频 < 30 分钟
- AI 出题响应时间：单个考点 < 30 秒，3 个考点 < 60 秒
- 考点抽取响应时间：单个课件 < 5 分钟
- C 端 ChatBot 检索响应时间：< 3 秒

### 5.2 安全需求

- 用户通过 NIO SSO 登录，Token 有效期 24 小时
- 培训师数据权限隔离，只能查看和操作自己创建的数据
- 课件存储加密，访问需鉴权

### 5.3 可用性需求

- 系统可用性 > 99.5%
- 飞书机器人消息推送成功率 > 99%
