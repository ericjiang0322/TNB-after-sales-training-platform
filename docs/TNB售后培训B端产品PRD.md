# TNB 售后学练平台 B 端产品需求文档

## 1. 需求背景

### 1.1 业务背景

TNB 售后体系覆盖服务代表、机电技师、钣金技师、喷漆技师、保修员、配附件专员、车间负责人、技术负责人、质检员等多个岗位，每个岗位分初级、中级、高级、管理四个认证等级。目前售后培训依赖线下培训和人工出题，存在以下问题：

- 课件分散在 PDF、PPT 等格式中，学员检索困难
- 考点和题目依赖人工维护，更新效率低
- 不同岗位、不同等级的培训内容缺乏系统化关联
- 学员练习数据无法追踪，培训效果难以衡量

### 1.2 产品定位

TNB 售后学练平台是面向售后培训场景的智能化学习练习系统，通过 AI 技术实现课件知识切片、考点自动抽取、智能出题，支持岗位课程（认证等级导向）和专项课程（知识模块导向）两种培训模式，为售后人员提供个性化的练习推送和学习效果追踪。

### 1.3 目标用户

| 角色 | 职责 | 使用频率 |
|------|------|----------|
| 系统管理员 | 维护能力维度、岗位课程、系统配置 | 低 |
| 培训师 | 维护专项课程、上传课件、抽取考点、AI 出题、审核发布 | 高 |
| 学员 | 接收推送练习、自主练习、查看练习记录 | 高 |

### 1.4 项目范围

**MVP 范围**：专项课程完整闭环（课程创建 → 课件上传 → 知识切片 → 考点抽取 → AI 出题 → 人工审核 → 发布 → 学员练习 → 数据查看）。岗位课程在 MVP 中仅展示后台已维护的结果，支持题目微调，暂不支持重新出题。

## 3. 功能需求详述

### 3.1 岗位管理

#### 3.1.1 功能概述

维护售后体系中的岗位信息，包括岗位名称、岗位类型（技术类/非技术类）、认证等级 Level 顺序、认证 Code 映射关系。岗位数据从 `dwd_so_welearn_certificate_info_1d_f` 数仓表同步，业务可在 B 端对岗位、认证等级、认证 Code 进行维护调整。

#### 3.1.2 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 岗位 ID | 字符串 | 系统生成 | 唯一标识 |
| 岗位名称 | 字符串 | 是 | 如"服务代表"、"机电技师" |
| 岗位类型 | 枚举 | 是 | 技术类 / 非技术类 |
| 排序 | 整数 | 是 | 展示顺序 |
| 创建时间 | 时间 | 系统自动 | |

**认证等级定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 认证等级 ID | 字符串 | 系统生成 | 唯一标识 |
| 所属岗位 ID | 字符串 | 是 | 关联岗位 |
| 等级名称 | 枚举 | 是 | 初级 / 中级 / 高级 / 管理 |
| Level | 整数 | 是 | 等级顺序值，如初级=1、中级=2、高级=3、管理=4 |
| 认证 Code | 字符串 | 是 | 如 JSR-C、ISR-C、SSR-C、SRM-C |
| 排序 | 整数 | 是 | 同岗位内的展示顺序，按 Level 升序 |

#### 3.1.3 业务逻辑

- 岗位列表：展示所有岗位，支持按岗位类型筛选
- 岗位维护：新增、编辑岗位名称、岗位类型
- 认证等级维护：为每个岗位配置认证等级及对应的认证 Code
  - Level 值决定 C 端展示顺序，Level 越高展示越靠后
  - 一个岗位至少有一个认证等级，最多四个（初级/中级/高级/管理）
  - 认证 Code 需唯一，不可重复
- 岗位删除：岗位下无关联课程时才允许删除
- 认证等级删除：该等级下无关联课程时才允许删除

#### 3.1.4 与课程的关联

- 岗位课程的创建需选择所属岗位 + 认证等级
- C 端根据用户当前岗位的认证等级，按 Level 顺序展示对应课程
- 用户目标认证等级默认为其当前岗位 Level 的上一级（如当前初级，目标中级）

### 3.2 课程管理

#### 3.2.1 功能概述

课程分为**岗位课程**和**专项课程**两类，分开维护。岗位课程面向认证等级，一个认证等级对应一个课程；专项课程面向知识模块，区分技术类和非技术类。

#### 3.2.2 岗位课程

**业务逻辑**：
- 岗位课程数据从现有系统同步，管理员后台维护
- 一个岗位等级对应一个课程（如 JSR 服务代表初级对应一个课程）
- 课程包含多个能力维度，每个能力维度包含多个知识模块
- **维护分工**：
  - 课件特别多特别大、切片容易出问题的岗位课程 → 由研发侧处理
  - 其他岗位课程 → 由业务自行维护（上传课件、抽取考点、AI 出题）
- 岗位课程在 MVP 中仅展示结果，不支持重新出题，但支持题目手动微调

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 课程 ID | 字符串 | 系统生成 | 唯一标识 |
| 课程名称 | 字符串 | 是 | 如"服务代表初级（JSR）" |
| 岗位类型 | 枚举 | 是 | 技术类 / 非技术类 |
| 岗位名称 | 枚举 | 是 | 服务代表、机电技师等 |
| 认证等级 | 枚举 | 是 | 初级 / 中级 / 高级 / 管理 |
| 认证 Code | 字符串 | 是 | 如 JSR-C、ISR-C |
| 课程状态 | 枚举 | 是 | 上架 / 下架 |
| 维护方 | 枚举 | 是 | 业务维护 / 研发维护 |
| 创建时间 | 时间 | 系统自动 | |
| 更新时间 | 时间 | 系统自动 | |

**交互逻辑**：
- 岗位课程列表页：展示所有岗位课程，支持按岗位类型、岗位名称、认证等级、维护方筛选
- 课程详情页：展示课程下的能力维度 → 知识模块 → 考点 → 题库
  - 业务维护的课程：支持完整操作（上传课件、抽取考点、AI 出题、题目微调）
  - 研发维护的课程：仅展示结果，不支持重新出题，但支持题目手动微调
- 新建岗位课程：选择岗位 + 认证等级，自动带出认证 Code，选择维护方

#### 3.2.3 专项课程

**业务逻辑**：
- 专项课程由培训师创建和维护
- 一个专项课程只有一个能力维度（"知识掌握"），课程下直接挂知识模块
- 专项课程区分技术类和非技术类
- 课程下架后，C 端该课程的练习也下架

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 课程 ID | 字符串 | 系统生成 | 唯一标识 |
| 课程名称 | 字符串 | 是 | 如"ES9 新车型技术培训" |
| 课程类型 | 枚举 | 是 | 技术类 / 非技术类 |
| 课程状态 | 枚举 | 是 | 上架 / 下架 |
| 创建人 | 字符串 | 系统自动 | 培训师域账号 |
| 创建时间 | 时间 | 系统自动 | |
| 更新时间 | 时间 | 系统自动 | |

**交互逻辑**：
- 专项课程列表页：展示培训师有权限的专项课程，支持按课程类型、状态筛选
- 新建课程：填写课程名称、选择课程类型（技术类/非技术类）
- 编辑课程：修改课程名称、课程类型、上下架状态
- 课程详情页：展示课程下的知识模块列表，可进入知识模块管理

#### 3.2.4 页面原型

```
+--------------------------------------------------+
| 课程管理                              [+ 新建课程] |
+--------------------------------------------------+
| 标签页: [岗位课程] [专项课程]                      |
+--------------------------------------------------+
| 筛选: [岗位类型: 全部 ▼] [岗位: 全部 ▼] [等级: 全部 ▼] [维护方: 全部 ▼] |
+--------------------------------------------------+
| 课程名称          | 岗位类型 | 岗位     | 等级   | 维护方   | 状态   | 操作 |
|-------------------|----------|----------|--------|----------|--------|------|
| 服务代表初级(JSR) | 非技术类 | 服务代表 | 初级   | 业务维护 | 上架   | 查看 |
| 机电技师中级(ST)  | 技术类   | 机电技师 | 中级   | 研发维护 | 上架   | 查看 |
| ...               |          |          |        |          |        |      |
+--------------------------------------------------+
```

### 3.3 能力维度管理

#### 3.3.1 功能概述

能力维度是岗位课程的特有层级，一个岗位课程包含多个能力维度（如"L1-专业知识"、"L2-标准服务流程执行"），每个能力维度包含多个知识模块。能力维度由管理员后台维护，业务可自行调整，但调整时需处理关联考点的迁移。

#### 3.3.2 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 能力维度 ID | 字符串 | 系统生成 | 唯一标识 |
| 能力维度名称 | 字符串 | 是 | 如"L1-专业知识" |
| 能力维度描述 | 文本 | 否 | 能力要求说明 |
| 所属课程 ID | 字符串 | 是 | 关联岗位课程 |
| 排序 | 整数 | 是 | 展示顺序 |

#### 3.3.3 交互逻辑

- 能力维度列表：按课程分组展示能力维度
- 新建/编辑：管理员维护能力维度名称、描述、排序
- 删除约束：能力维度下无知识模块时才允许删除
- **调整约束（考点迁移）**：
  - 能力维度下存在考点时，不允许直接删除该能力维度
  - 如需调整能力维度归属（如从一个课程调整到另一个课程），需先将该能力维度下的所有考点和考题迁移到新的能力维度/课程上，迁移完成后方可下架/删除原能力维度
  - 迁移流程：选择目标能力维度 → 确认迁移考点和考题 → 执行迁移 → 原能力维度下架/删除

### 3.4 知识模块管理

#### 3.4.1 功能概述

知识模块是课程下的学习单元，对应具体的课件集合。

**岗位课程**：知识模块挂在能力维度下，一个能力维度包含多个知识模块。
**专项课程**：知识模块直接挂在课程下，课程只有一个能力维度（"知识掌握"）。

#### 3.4.2 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 知识模块 ID | 字符串 | 系统生成 | 唯一标识 |
| 知识模块名称 | 字符串 | 是 | 如"车辆解锁、上锁和门窗控制" |
| 所属课程 ID | 字符串 | 是 | 关联课程 |
| 所属能力维度 ID | 字符串 | 岗位课程必填 | 关联能力维度 |
| 排序 | 整数 | 是 | 展示顺序 |
| 创建时间 | 时间 | 系统自动 | |

#### 3.4.3 交互逻辑

- 知识模块列表：按课程（或能力维度）分组展示
- 新建/编辑：维护知识模块名称、排序
- 删除：知识模块下无课件时才允许删除
- 进入知识模块：可管理课件、考点、题库

### 3.5 课件知识管理

#### 3.5.1 功能概述

支持 PDF、PPT 两种格式的课件上传，系统进行知识切片和向量化存储，供 C 端 ChatBot 检索使用。

#### 3.5.2 课件上传

**支持的格式**：
- PDF：直接解析文本内容
- PPT：解析每页文本内容

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 课件 ID | 字符串 | 系统生成 | 唯一标识 |
| 课件名称 | 字符串 | 是 | 原始文件名或自定义名称 |
| 课件类型 | 枚举 | 是 | PDF / PPT |
| 总页数 | 整数 | 系统自动 | PDF/PPT 解析得到的总页数 |
| 已处理页数 | 整数 | 系统自动 | 当前已切片完成的页数 |
| 所属知识模块 ID | 字符串 | 是 | 关联知识模块 |
| 文件 URL | 字符串 | 是 | 存储地址 |
| 切片状态 | 枚举 | 系统自动 | 待切片 / 切片中 / 已切片 / 切片失败 |
| 切片进度 | 字符串 | 系统自动 | 如"已处理 45/120 页"，基于页数计算 |
| 创建时间 | 时间 | 系统自动 | |

#### 3.5.3 知识切片

**切片策略**：
- PDF：按页切片 + AI 语义切片结合
- PPT：按页切片

**切片字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 切片 ID | 字符串 | 唯一标识 |
| 所属课件 ID | 字符串 | |
| 切片内容 | 文本 | 文本内容 |
| 切片类型 | 枚举 | 按页 / 按章节 / 语义段落 |
| 页码 | 字符串 | PDF/PPT 页码 |
| 向量 Embedding | 向量 | 向量化表示 |

**交互逻辑**：
- 课件上传后，系统异步进行切片和向量化
- 切片过程中，系统实时记录已处理页数，计算切片进度
- 切片进度以"已处理页数/总页数"的形式展示，如"已处理 45/120 页"
- 课件列表中展示切片状态 + 切片进度，方便用户了解大课件的处理进度
- 切片完成后，可查看切片列表和切片内容
- 切片失败的课件可重新触发切片，重新触发后重置进度
- C 端 ChatBot 基于向量化存储的切片内容回答学员问题

### 3.6 考点管理

#### 3.6.1 功能概述

考点是针对一个知识点的标准问答/概念解释，从课件切片中 AI 自动抽取，经人工审核后用于 AI 出题。AI 抽取调用 Langfuse 上的 `knowledge-point-extract` prompt，抽取后自动调用 `knowledge-point-merge` 合并去重，最终输出包含多维度评分信息的结构化考点。

#### 3.6.2 考点抽取

**业务逻辑**：
- 课件切片完成后，管理员/培训师可选择以下方式触发考点抽取：
  - **立即抽取**：手动点击"开始抽取考点"，立即执行
  - **定时抽取**：配置定时任务（如"每天凌晨 2:00"），系统自动执行大课件或多个课件的考点批量抽取
- 系统调用 `knowledge-point-extract` prompt 从切片中提取考点
- 提取完成后自动调用 `knowledge-point-merge` 对不同切片间语义相近的考点合并去重
- 抽取的考点进入"待审核"状态
- 一个考点只来源于一个切片（合并后保留原始来源切片集合）
- 考点继承所属课程的岗位类型（技术类/非技术类）
- **定时任务适用场景**：大课件（页数 > 100 页）或多个课件批量抽取，避免高峰期占用资源，利用非工作时间自动执行

**考点字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 考点 ID | 字符串 | 系统生成 | 唯一标识 |
| 考点文本 | 文本 | 是 | 知识点描述，控制在 15-120 字 |

| 重要度评分 | 整数 0-100 | 是 | AI 评估的考点重要程度 |
| 信息密度评分 | 整数 0-100 | 是 | AI 评估的信息量密度 |
| 适题性评分 | 整数 0-100 | 是 | AI 评估的出题适用性 |
| 业务风险评分 | 整数 0-100 | 是 | AI 评估的答错业务风险 |
| 支持题型 | 数组 | 是 | 该考点可出的题型：单选题 / 多选题 / 判断题 |

#### 3.6.3 考点审核

**交互逻辑**：
- 考点列表：展示待审核、已通过、已拒绝的考点，支持按状态、考点类型、支持难度筛选
- 列表展示：考点文本、考点类型、各维度评分（重要度/信息密度/适题性/业务风险）、支持题型、支持难度、来源切片数量、原文依据
- 评分展示：以进度条或色标形式展示 0-100 分，培训师可快速判断考点质量
- 审核操作：
  - 通过：考点状态变为"已通过"，可用于 AI 出题
  - 拒绝：考点状态变为"已拒绝"，可填写拒绝原因
  - 编辑：修改考点文本、考点类型、各维度评分、支持题型、支持难度
- 批量审核：支持批量通过/拒绝
- 来源切片联查：点击来源切片数量可查看具体切片内容，辅助判断考点合理性

#### 3.6.4 考点迁移

**业务逻辑**：
- 当能力维度或课程需要调整时（如能力维度归属变更、课程 restructuring），需先处理关联考点的迁移
- **迁移约束**：
  - 考点必须先挂载到新的能力维度/课程上，才能下架/删除原能力维度或课程
  - 迁移时，考点关联的考题（已发布和未发布）一并迁移到新的知识模块下
  - 迁移过程中，原考点和考题保持可用状态，直到迁移完成确认
- **迁移流程**：
  1. 选择需要迁移的考点（支持批量选择）
  2. 选择目标能力维度/知识模块
  3. 确认迁移：展示迁移影响范围（考点数量、关联考题数量、已发布题目数量）
  4. 执行迁移：更新考点和考题的所属关系
  5. 确认原能力维度/课程可下架/删除

### 3.7 AI 策略配置

#### 3.7.1 功能概述

Prompt 实际维护在 Langfuse 平台（`training-agent` 项目），包含 `knowledge-point-extract`（考点抽取）、`knowledge-point-merge`（考点合并去重）、`question-generation`（出题）、`question-review`（题目审核）四个独立 prompt。B 端不维护 prompt 文本，仅维护调用参数模板，即"策略配置"——绑定 Langfuse prompt 名称，并预设默认的调用参数组合。

#### 3.7.2 策略用途

| 用途 | 绑定 Langfuse Prompt | 说明 |
|------|---------------------|------|
| 考点抽取 | `knowledge-point-extract` | 从课件切片中提取考点，输入参数为切片内容 |
| 考点合并 | `knowledge-point-merge` | 考点抽取后自动调用，合并语义相近的考点去重 |
| 出题 | `question-generation` | 根据考点生成题目，输入参数含岗位类型、难度、题型、图片模式 |
| 题目审核 | `question-review` | 预留，AI 辅助预审核题目质量 |

#### 3.7.3 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 策略 ID | 字符串 | 系统生成 | 唯一标识 |
| 策略名称 | 字符串 | 是 | 如"技术类-新车型出题策略" |
| 策略用途 | 枚举 | 是 | 考点抽取 / 出题 |
| 适用岗位类型 | 枚举 | 是 | 技术类 / 非技术类 |
| 关联 Langfuse Prompt | 字符串 | 是 | 如 `question-generation` |
| 默认题型 | 数组 | 否 | 单选题 / 多选题 / 判断题，出题时可选 |
| 默认难度 | 数组 | 否 | 易 / 中 / 难，出题时可选 |
| 默认图片模式 | 枚举 | 否 | 自动 / 优先有图 / 必须有图 / 禁止配图 |
| 创建人 | 字符串 | 系统自动 | |
| 创建时间 | 时间 | 系统自动 | |

#### 3.7.4 交互逻辑

- 策略列表：按用途和适用岗位类型分组展示
- 新建/编辑：维护策略名称、用途、适用岗位类型、关联 Langfuse Prompt、默认参数
- 删除：策略未被使用时允许删除
- 出题时直接选用策略，自动填充默认题型、难度、图片模式
- 岗位课程的策略编辑权限仅给管理员
- 培训师可维护专项课程的策略配置
- 实际的 Prompt 文本在 Langfuse 平台维护，B 端不展示也不编辑

### 3.8 AI 出题管理

#### 3.8.1 功能概述

基于已审核通过的考点，选择 AI 策略配置，调用 Langfuse 上的 `question-generation` prompt 自动生成题目，经人工审核后发布到题库。

#### 3.8.2 出题流程

```
选择考点（最多 3 个）→ 选择 AI 策略（自动填充默认参数）→ 设置出题参数 → AI 生成题目 → 人工审核 → 发布到题库
```

**业务规则**：
- 每次最多选择 3 个考点
- 每个考点只能出 1 道题
- 题型固定：单选题、多选题、判断题
- 难度可定义：易、中、难（在 Langfuse prompt 中约束）
- 审核不通过的题目可直接删除
- 已发布的题目不能删除，只能失效

#### 3.8.3 AI 出题输入参数

调用 `question-generation` prompt 时，B 端需要组装以下参数：

| 参数 | B 端中文名称 | 来源/规则 |
|------|-------------|----------|
| `positionType` | 岗位类型 | 继承课程类型（技术类/非技术类），映射为 TECHNICAL / NON_TECHNICAL |
| `difficulty` | 难度 | 用户选择：易= EASY / 中= MEDIUM / 难= HARD |
| `questionType` | 题型 | 用户选择：单选题= SINGLE_CHOICE / 多选题= MULTIPLE_CHOICE / 判断题= TRUE_FALSE |
| `imageMode` | 图片模式 | 用户选择：自动 / 优先有图 / 必须有图 / 禁止配图 |
| `knowledgePointId` | 考点 ID | 所选考点 ID |
| `pointText` | 考点文本 | 所选考点的考点文本 |
| `chunks` | 原文切片 | 根据考点来源切片组装原文内容 |

**图片模式说明**：

| B 端展示 | 映射值 | 说明 |
|----------|--------|------|
| 自动 | AUTO | 有合格图片时自动使用，无合格图片则无图 |
| 优先有图 | PREFERRED | 优先使用合格图片，无合格图片则无图 |
| 必须有图 | REQUIRED | 必须使用至少一张合格图片，无合格图片时标记题目异常 |
| 禁止配图 | FORBIDDEN | 禁止使用任何图片 |

#### 3.8.4 题目字段

| 字段 | B 端中文名称 | 类型 | 必填 | 说明 |
|------|-------------|------|------|------|
| 题目 ID | 字符串 | 系统生成 | 唯一标识 |
| questionMd | 题干 | 文本 | 是 | 题目内容，支持 Markdown 格式 |
| questionType | 题型 | 枚举 | 是 | 单选题 / 多选题 / 判断题 |
| options | 选项 | JSON | 是 | 每个选项含：选项标识（A/B/C/D）、选项内容（Markdown）、配图 |
| correctAnswer | 正确答案 | 字符串/数组 | 是 | 单选/判断为字符串，多选为数组 |
| analysisMd | 解析 | 文本 | 是 | 答案解析，支持 Markdown 格式 |
| images | 题目配图 | 数组 | 否 | 题目级图片 URL 列表 |
| knowledgePointId | 关联考点 ID | 字符串 | 是 | 关联的考点 |
| difficulty | 难度 | 枚举 | 是 | 易 / 中 / 难 |
| knowledgeModuleId | 所属知识模块 ID | 字符串 | 是 | |
| status | 状态 | 枚举 | 是 | 待审核 / 已通过 / 已发布 / 已失效 |
| createTime | 创建时间 | 时间 | 系统自动 | |

#### 3.8.5 审核与微调

**交互逻辑**：
- AI 生成的题目进入"待审核"状态
- 审核操作：
  - 通过：题目状态变为"已通过"，可继续发布
  - 删除：直接删除题目
  - 编辑：修改题干、选项内容、选项配图、解析（不可修改题型）
- 重新出题：对不满意的题目，可调整策略参数后重新生成
- 发布：审核通过的题目发布到对应知识模块的题库

### 3.9 题库管理

#### 3.9.1 功能概述

- 用户已练习的数据快照了题目 ID
- 题目不需要版本管理
- 岗位课程的题目支持手动微调（编辑题干、选项内容、选项配图、解析）

- 题目详情展开：展示题干（Markdown 渲染）、选项、配图、解析
- 编辑：修改题干、选项内容、选项配图、解析（仅未发布或本模块内的题目）
- 取消发布：已发布的题目可取消发布，状态变为"已失效"
- 重新发布：失效的题目可重新审核后发布

### 3.12 定时任务管理

#### 3.12.1 功能概述

支持对切片和考点抽取配置定时任务，解决大课件切片和考点抽取耗时长、占用高峰期资源的问题。利用非工作时间（如凌晨）自动执行，并支持任务执行进度追踪。

#### 3.12.2 任务类型

| 任务类型 | 说明 | 触发时机 |
|----------|------|----------|
| 切片任务 | 对指定课件执行知识切片 | 课件上传后定时触发或手动配置 |
| 考点抽取任务 | 对已切片的课件执行考点抽取 | 切片完成后定时触发或手动配置 |
| 批量切片任务 | 对多个课件批量执行切片 | 按配置时间批量执行 |
| 批量考点抽取任务 | 对多个已切片课件批量抽取考点 | 按配置时间批量执行 |

#### 3.12.3 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 任务 ID | 字符串 | 系统生成 | 唯一标识 |
| 任务名称 | 字符串 | 是 | 如"ES9 大课件凌晨切片任务" |
| 任务类型 | 枚举 | 是 | 切片任务 / 考点抽取任务 / 批量切片 / 批量考点抽取 |
| 关联对象 | 数组 | 是 | 关联的课件 ID 或知识模块 ID |
| 定时表达式 | 字符串 | 是 | Cron 表达式，如 `0 2 * * *`（每天凌晨 2:00） |
| 任务状态 | 枚举 | 系统生成 | 待执行 / 执行中 / 已完成 / 失败 / 已暂停 |
| 下次执行时间 | 时间 | 系统自动 | 根据 Cron 表达式计算 |
| 创建人 | 字符串 | 系统自动 | |
| 创建时间 | 时间 | 系统自动 | |

#### 3.12.4 执行记录

每次任务执行生成一条执行记录：

| 字段 | 类型 | 说明 |
|------|------|------|
| 执行 ID | 字符串 | 唯一标识 |
| 所属任务 ID | 字符串 | |
| 开始时间 | 时间 | |
| 结束时间 | 时间 | |
| 执行状态 | 枚举 | 成功 / 失败 / 执行中 |
| 处理对象数 | 整数 | 本次处理了多少个课件/知识模块 |
| 处理页数 | 整数 | 切片任务专用，本次处理的总页数 |
| 完成页数 | 整数 | 切片任务专用，本次成功处理的页数 |
| 日志 | 文本 | 执行日志，便于排查失败原因 |

#### 3.12.5 交互逻辑

- 任务列表：展示所有定时任务，支持按任务类型、状态筛选
- 新建任务：选择任务类型 → 选择关联对象（课件/知识模块）→ 配置 Cron 表达式 → 保存
- 任务启停：支持手动启用/暂停定时任务
- 执行记录：点击任务可查看历史执行记录列表
- 执行详情：查看单次执行的详细日志、处理进度、失败原因
- 失败重试：失败的执行支持一键重试

#### 3.12.6 进度展示

**切片任务进度**：
- 任务执行中，实时展示"已处理页数 / 总页数"
- 以进度条形式直观展示
- 每个课件单独展示进度

**考点抽取任务进度**：
- 任务执行中，展示"已处理切片数 / 总切片数"
- 每个知识模块单独展示进度

### 3.13 系统配置

#### 3.13.1 用户登录

- 通过 NIO SSO 登录，首次登录自动注册为 TNB 售后学练平台用户
- C 端首次登录时选择自己的岗位，默认该岗位 Level 最低的一级（初级）

#### 3.13.2 角色权限

| 角色 | 数据权限 | 功能权限 |
|------|----------|----------|
| 系统管理员 | 全部数据 | 全部功能 |
| 培训师 | 仅自己创建的数据 | 专项课程管理、课件、考点、出题、发布 |

**功能权限控制**：
- 通过 Mate 系统后台的功能权限点控制
- 具体权限点由产品经理补充配置

#### 3.13.3 岗位认证匹配规则

**数据来源**：`dwd_so_welearn_certificate_info_1d_f` 数仓表

**匹配规则**：
- 根据用户认证 Code 的 prefix 匹配当前岗位认证等级表，确定用户当前岗位的认证等级
- 同一岗位存在多次认证时，取生效时间最近的一次

**认证等级判定逻辑**：
- 最近生效结束时间 < 当前时间：取该岗位的上一级为当前认证等级
- 最近生效结束时间 >= 当前时间：取本认证等级
- 如存在不同岗位认证，则该用户为多岗位用户
- 如不存在任何岗位认证记录：用户手动选择岗位，目标等级默认为该岗位 Level 最低的一级

**示例**：

| 岗位 | 初级 | 中级 | 高级 | 管理 |
|------|------|------|------|------|
| 服务代表 | JSR-C | ISR-C | SSR-C | SRM-C |
| 机电技师 | MT-C | ST-C | DT-C | - |
| 钣金技师 | BAT-C | BST-C | BET-C | - |
| 喷漆技师 | PAT-C | PST-C | PET-C | - |
| 保修员 | WPL1-T | WPL2-C | - | - |
| 配附件专员 | PAS26-T | - | - | - |
| 车间负责人 | - | - | - | WMT-T |
| 技术负责人 | - | - | - | TMT-T |
| 质检员 | - | - | - | QIT-T |

**C 端展示规则**：
- 按用户当前岗位的认证等级 Level 顺序展示课程，Level 越高展示越靠后
- 用户目标认证等级默认为当前 Level 的上一级（如当前初级 Level=1，目标中级 Level=2）
- 多岗位用户按各岗位分别展示对应课程

## 4. 数据模型

### 4.1 核心实体关系

```
岗位 (Position)
  ├── 岗位类型: 技术类 / 非技术类
  ├── 岗位名称: 服务代表、机电技师...
  └── 认证等级 (CertLevel)
      ├── Level: 1=初级 / 2=中级 / 3=高级 / 4=管理
      └── 认证 Code: JSR-C、ISR-C、MT-C...

课程 (Course)
  ├── 岗位课程: 关联岗位 + 认证等级 + 维护方(业务/研发)
  └── 专项课程: 课程类型(技术类/非技术类)

能力维度 (Competency) [仅岗位课程]
  └── 关联课程

知识模块 (KnowledgeModule)
  ├── 岗位课程: 关联能力维度
  └── 专项课程: 直接关联课程

课件 (Courseware)
  └── 关联知识模块

切片 (Slice)
  └── 关联课件

考点 (ExamPoint)
  ├── 考点类型: 概念定义 / 参数标准 / 流程步骤 / 规范要求 / 故障诊断 / 业务场景
  ├── 评分: 重要度 / 信息密度 / 适题性 / 业务风险 (0-100)
  ├── 关联切片 + 知识模块
  └── 支持题型 + 支持难度

AI 策略配置 (AIStrategy)
  ├── 用途: 考点抽取 / 出题
  ├── 适用岗位类型: 技术类 / 非技术类
  └── 关联 Langfuse Prompt 名称

题目 (Question)
  ├── 题干 (questionMd) + 解析 (analysisMd): Markdown 格式
  ├── 选项: 含选项标识、内容(Markdown)、配图
  ├── 关联考点 + 知识模块
  └── 图片模式 + 题目配图

发布配置 (PublishConfig)
  └── 关联课程 + 知识模块 + 发布范围

定时任务 (ScheduledTask)
  ├── 任务类型: 切片任务 / 考点抽取任务 / 批量切片 / 批量考点抽取
  ├── 关联对象: 课件 ID / 知识模块 ID 列表
  ├── 定时表达式: Cron 表达式
  └── 任务状态: 待执行 / 执行中 / 已完成 / 失败 / 已暂停

任务执行记录 (TaskExecution)
  ├── 关联定时任务
  ├── 执行状态: 成功 / 失败 / 执行中
  ├── 处理页数 / 完成页数（切片任务）
  └── 执行日志

练习记录 (PracticeRecord)
  └── 关联用户 + 题目 + 课程 + 知识模块
```

## 5. 非功能需求

### 5.1 性能需求

- 课件上传后切片完成时间：PDF/PPT < 5 分钟
- AI 出题响应时间：单个考点 < 30 秒，3 个考点 < 60 秒
- 考点抽取响应时间：单个课件 < 5 分钟
- C 端 ChatBot 检索响应时间：< 3 秒

### 5.2 安全需求

- 用户通过 NIO SSO 登录，Token 有效期 24 小时
- 培训师数据权限隔离，只能查看和操作自己创建的数据
- 课件存储加密，访问需鉴权

### 5.3 可用性需求

- 系统可用性 > 99.5%
- 飞书机器人消息推送成功率 > 99%
